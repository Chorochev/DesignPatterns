	Строитель (англ. Builder) 
	— порождающий шаблон проектирования предоставляет способ создания составного объекта.
-------------------------------------------------------------------------------------------
Цель
	Отделяет конструирование сложного объекта от его представления, так что в результате 
	одного и того же процесса конструирования могут получаться разные представления.
-------------------------------------------------------------------------------------------
Плюсы
	- позволяет изменять внутреннее представление продукта;
	- изолирует код, реализующий конструирование и представление;
	- дает более тонкий контроль над процессом конструирования.
-------------------------------------------------------------------------------------------
Применение
	- алгоритм создания сложного объекта не должен зависеть от того, из каких частей состоит 
	объект и как они стыкуются между собой;
	- процесс конструирования должен обеспечивать различные представления конструируемого объекта.
-------------------------------------------------------------------------------------------

Участники
	Product: представляет объект, который должен быть создан. В данном случае все части объекта заключены в списке parts.
	Builder: определяет интерфейс для создания различных частей объекта Product
	ConcreteBuilder: конкретная реализация Buildera. Создает объект Product и определяет интерфейс для доступа к нему
	Director: распорядитель - создает объект, используя объекты Builder
